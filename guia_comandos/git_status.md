# `git status`

Voltar ao Guia de Comandos

## O que faz?

O comando `git status` tem a função de mostrar o estado atual do seu repositório. Ele lista quais arquivos foram modificados no seu working directory, quais estão na staging area (prontos para o commit) e quais ainda não estão sendo rastreados pelo Git (untracked). É o comando de "olhar ao redor" antes de agir.

## Formas de usar

- `git status`: exibe o status completo e detalhado;
- `git status -s`: exibe o status de forma resumida (short), ideal para quando há muitas alterações.

## Formas de usar (pelo GitHub Desktop)

Pelo programa, você não precisa rodar um comando. O status é a própria interface principal: o painel à esquerda (aba "Changes") lista automaticamente todos os arquivos modificados, adicionados ou deletados.

## Formas de desfazer

Não se aplica. Como é um comando apenas de leitura (inspeção), ele não altera nada no repositório, logo não há o que desfazer.

Palavras chaves

- `estado`, `verificar`, `mudanças`, `inspecionar`
