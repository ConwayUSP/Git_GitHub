# `git log`

Voltar ao [Guia de Comandos](README.md) 

## O que faz?

O comando `git log` tem a função de exibir o histórico de commits do repositório. Ele mostra quem fez as alterações, quando elas foram feitas, a mensagem do commit e o código hash (ID único) de cada um. É a "máquina do tempo" do seu projeto.

Formas de usar

- `git log`: exibe o histórico completo de commits, do mais recente para o mais antigo;
- `git log --oneline:` exibe o histórico de forma resumida (um commit por linha), mostrando apenas o hash curto e a mensagem;
- `git log -p`: exibe o histórico com as diferenças exatas de código adicionado ou removido em cada commit.

## Formas de usar (pelo GitHub Desktop)

Pelo programa, acesse a aba History localizada no painel da esquerda (ao lado de "Changes"). Lá você verá a lista cronológica e visual de todos os commits.

## Formas de desfazer

Não se aplica. É um comando de visualização e não altera nada. Para sair da tela do log no terminal, pressione a tecla q.

## Palavras chaves
