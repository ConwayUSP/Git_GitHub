# Fork

Voltar para o [Glossário](README.md).

Um **Fork** é uma _bifurcação_ de um repositório remoto para criar outro repositório remoto, o que difere de clonar, que cria um repositório local. Os forks são usados para trabalhar em ideias antes de propor alterações no repositório original, uma vez que em muitos projetos, como os _Open Source_, usuários não tem permissão de edição. Além disso, desenvolvedores criam forks para "brincar", testar ou experimentar com um projeto real, podendo até criar novos projetos a partir deles, exemplos famosos são os muitos forks do navegador Chrome (Chromium, Thorium, Brave, etc) e Firefox (LibreWolf, Zen, Mercury, etc).

Através de plataformas de hospedagem de repositórios, você pode "forkar" um projeto, mantê-lo em sincroniza com o original, propor alterações ou dar o seu próprio rumo de desenvolvimento. Mas é importante ressaltar que muitos projetos de código aberto, possuem **licenças** legais, explicitando o que você pode fazer ao "forkar" o projeto, como uso comercial, por exemplo, então é sempre bom ler a licença!

> **TLDR;** Um Fork é uma bifurcação de um repositório remoto, pode ser usado para teste e experimentação, ou para criar um projeto novo.
