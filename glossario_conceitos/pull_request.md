# Pull Request

Voltar para o [Glossário](README.md).

O **Pull Request** (PR) é basicamente uma requisição ou um pedido para incluir mudanças em um repositório. Imagine que você clonou um repositório, criou uma branch, fez alterações e quer enviá-las para a branch principal. Acontece que existe uma pessoa responsável pelo funcionamento da branch principal, em respeito (ou obrigação) a essa pessoa, você abre um pedido formal para que ela inclua a sua branch na principal. Isso é basicamente o processo de abrir uma _pull request_.

Em uma pull request, **revisor** (a pessoa que aprova ou bloqueia) e o **colaborador** (autor da pull) podem discutir ou colaborar entre si para que a PR seja aceita. Através de plataformas como o Github e outras, as pull requests podem ser tratadas como tarefas do planejamento de um time, marcos para novas versões de um produto, ou como fóruns para discussão sobre uma funcionalidade específica. Inclusive, essas plataformas permitem que scripts de teste, de qualidade de software ou de _deploy_ sejam executados para verificar se PR não prejudica o funcionamento do código principa. Desse jeito, Pull requests são uma pedra fundamental do desenvolvimento de software colaborativo.

> **TLDR;** Pull request é um pedido de _merge_ (inclusão) de uma branch em outra. Ficando a cargo de um revisor aprovar ou negar a PR.
